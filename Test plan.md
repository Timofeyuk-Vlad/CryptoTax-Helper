## 1 Введение
Данный документ описывает план тестирования веб-сервиса "CryptoTax Helper". 
Документ предназначен для команды тестировщиков и разработчиков проекта. 
Цель тестирования - проверка соответствия реального поведения приложения ожидаемому поведению, описанному в требованиях к проекту.

## 2 Объект тестирования
В качестве объектов тестирования можно выделить следующие функциональные требования. Основные модули для тестирования:
- Аутентификация и безопасность
- Регистрация нового пользователя
- Двухфакторная аутентификация (2FA)
- Выбор страны резидентства (РФ/РБ)
- Подключение к криптобиржам через API
- Расчет налогов
- Генерация отчетов
- Экспорт в PDF и Excel

## 3 Атрибуты качества
1. Функциональность
Функциональная полнота: приложение должно выполнять все заявленные функции расчета налогов

2. Безопасность
Конфиденциальность: защита персональных и финансовых данных пользователей
Аутентичность: надежная проверка подлинности пользователей

3. Удобство использования
Интуитивность: интерфейс понятен пользователям без налоговых знаний

## 4 Риски
К рискам можно отнести:
- Некорректный расчет налогов из-за ошибок в алгоритмах или курсах валют
- Утечка конфиденциальных данных пользователей и API-ключей
- Проблемы с отображением на различных устройствах и браузерах

## 5 Аспекты тестирования
### Функциональное тестирование:
### 1. Регистрация и аутентификация
- Успешная регистрация с валидными данными
- Регистрация с уже существующим email
- Успешный вход с правильными учетными данными и 2FA
- 
### 2. Настройка налогового профиля
- Выбор страны резидентства (РФ/РБ)
- Валидация вводимых данных (ИНН, УНП)

### 3. Подключение к биржам
- Успешное подключение к Binance с валидными API-ключами
- Попытка подключения с невалидными API-ключами

### 4. Расчет налогов
- Применение метода FIFO для определения себестоимости

### 5. Генерация отчетов
- Экспорт отчетов в PDF с сохранением форматирования
- Экспорт данных в Excel с формулами и отдельными листами

### Нефункциональное тестирование:
### 1. Тестирование безопасности
- Шифрование персональных данных в базе
- Валидация входных данных

### 2. Тестирование производительности
- Время загрузки основных страниц
- Время генерации налогового отчета

### 3. Тестирование удобства использования
- Интуитивность интерфейса для новых пользователей

## 6 Подходы к тестированию
При тестировании будет использован ручной подход.
